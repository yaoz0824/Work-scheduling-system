# 自動打工排班最佳化系統 — 路由與頁面設計模板 (ROUTE_DESIGN)

本模板依據產品需求模板（PRD）、系統架構設計與資料庫設計，規劃本系統之 Flask 路由（Routes）架構。包含每個頁面的 URL 路徑、HTTP 方法、存取權限控管、Jinja2 模板清單與非同步 API 規格，作為 Controller 層的實作指南。

## 2. 路由詳細說明

### 2.1 身分驗證模組 (auth.py)
* **`GET /auth/login`**
  - **說明**：顯示系統登入頁。若使用者已登入，自動重導向至根路徑 `/`。
  - **輸出**：渲染 `auth/login.html`。
* **`POST /auth/login`**
  - **說明**：驗證使用者登入表單。
  - **輸入**：表單欄位 `username`（帳號）及 `password`（密碼）。
  - **處理邏輯**：
    1. 呼叫 `User.query.filter_by(username=username).first()`。
    2. 使用 `check_password_hash()` 檢查密碼。
    3. 驗證成功後，將 `user_id`、`username` 與 `role` 寫入 Flask `session`。
  - **輸出**：登入成功重導向至 `/`；失敗則帶入錯誤訊息並重新渲染 `auth/login.html`（回傳 HTTP 400）。
* **`POST /auth/logout`**
  - **說明**：登出當前帳號。
  - **處理邏輯**：清除 Flask `session` 內的所有資料。
  - **輸出**：重導向至 `/auth/login`。

### 2.2 首頁與儀表板模組 (main.py)
* **`GET /`**
  - **說明**：根路由分流控制器。
  - **處理邏輯**：檢查 `session.get('role')`，若為 `'manager'` 則重導向至 `/dashboard/manager`；若為 `'staff'` 則重導向至 `/dashboard/staff`。若未登入重導向至 `/auth/login`。
* **`GET /dashboard/manager`**
  - **說明**：店長後台控制中心。
  - **處理邏輯**：
    1. 呼叫 `Leave.get_all(status='Pending')` 取得待審假單數。
    2. 查詢今日日期區間內所有 `staff_id` 為 NULL 的 `shifts` 記錄（**當日缺工提示**）。
    3. 查詢今日已排定之班表，列出今日出勤員工。
  - **輸出**：渲染 `main/dashboard.html`，帶入上述統計變數。
* **`GET /dashboard/staff`**
  - **說明**：員工個人看板。
  - **處理邏輯**：
    1. 查詢該員工本週內所有已發佈的班表（`Shift.query.filter_by(staff_id=current_user, is_draft=False)`）。
    2. 呼叫 `Leave.get_leaves_by_user(current_user)` 取得個人請假審查歷史。
  - **輸出**：渲染 `main/dashboard.html`，帶入個人班表與假單變數。

### 2.3 排班管理模組 (schedule.py)
* **`GET /schedule/manage`**
  - **說明**：店長排班工作台，內嵌 FullCalendar 且開啟編輯權限。
  - **輸出**：渲染 `schedule/manage.html`。
* **`POST /schedule/auto`**
  - **說明**：執行一鍵智慧排班。
  - **輸入**：表單欄位 `start_date` 與 `end_date`（排班的日期區間）。
  - **處理邏輯**：
    1. 呼叫自動排班引擎服務：`scheduler_engine.run_auto_schedule(start_date, end_date)`。
    2. 演算法會自動避開員工「不可上班偏好」、「已核准請假」，並在不超工時上限前提下生成最佳排班陣列。
    3. 將生成的所有班次寫入 DB，狀態預設為 `is_draft=True`。
    4. 寫入 AuditLog 紀錄此次自動排班操作。
  - **輸出**：重導向回 `/schedule/manage`，日曆將呈現排班草稿。
* **`POST /schedule/publish`**
  - **說明**：正式發佈班表。
  - **輸入**：表單欄位 `start_date` 與 `end_date`。
  - **處理邏輯**：將該區間內所有班表的 `is_draft` 欄位更新為 `False`，使一般員工能正式查閱。

### 2.4 請假與代班模組 (leave.py)
* **`POST /leave/apply`**
  - **說明**：一般員工發起請假申請。
  - **輸入**：`start_time`（開始時間）、`end_time`（結束時間）、`leave_type`（假別）、`reason`（事由）。
  - **處理邏輯**：呼叫 `Leave.create()`。若發生重複請假衝突（`ValueError`），則拋出警告。
  - **輸出**：申請成功重導向回 `/dashboard/staff`。
* **`POST /leave/review/<id>/approve`**
  - **說明**：店長核准請假。
  - **處理邏輯**：呼叫 `leave.approve(reviewer_id=session['user_id'])`。
    - **重要副效能**：此方法內建自動處理機制，會自動將該請假時間範圍內，該員工已排定的班表之 `staff_id` 設為 NULL（變為空班/缺工），以觸發系統缺工提示。
  - **輸出**：重導向回 `/leave/review`。

### 2.5 AJAX APIs (api.py)
為了支援 FullCalendar 行事曆的非同步更新與即時合規預檢，設計了兩個高效 API：
* **`GET /api/shifts`**
  - **輸入**：查詢參數 `start`（開始 ISO 時間，如 `2026-06-01T00:00:00Z`）與 `end`（結束 ISO 時間）。
  - **處理邏輯**：呼叫 `Shift.get_shifts_by_range(start, end)`。如果使用者是 Staff，則過濾 `is_draft=True` 的班表（員工不可看草稿）。
  - **輸出**：回傳符合 FullCalendar 事件格式的 JSON 陣列：
    ```json
    [
      {
        "id": 1,
        "title": "早班 - 張小明",
        "start": "2026-06-01T08:00:00",
        "end": "2026-06-01T16:00:00",
        "color": "#28a745",
        "extendedProps": { "is_draft": false, "staff_id": 2 }
      }
    ]
    ```
* **`POST /api/shift/update`** (店長專用拖拉 API)
  - **輸入**：JSON Body：`{"shift_id": 1, "start_time": "...", "end_time": "...", "staff_id": 2}`。
  - **處理邏輯**：
    1. 呼叫 `Shift.get_by_id(shift_id)`。
    2. 呼叫 `Shift.check_scheduling_conflict()` 執行**動態防錯預檢**。
    3. 若檢核通過，寫入 DB，回傳 `{"status": "success"}`。
    4. 若檢核失敗（例如超時或請假衝突），則不寫入 DB，回傳警告資訊。
  - **輸出**：`{"status": "warning/error", "message": "違反單日最大工時上限限制"}`，HTTP 狀態碼為 200，讓前端 FullCalendar 能捕捉警告並將日曆區塊標示為警示高亮。

## 3. Jinja2 模板清單 (View)

本系統頁面渲染使用 Jinja2 模板，所有模板皆繼承 `base.html`：

1. **`templates/base.html`**：
   - **繼承關係**：最上層基礎版面。
   - **說明**：定義 RWD 響應式佈局。包含頂部 Navigation Bar（顯示商家標誌、登入者姓名與登出按鈕）與側邊選單 Sidebar（根據 `role` 動態呈現「排班管理」、「請假申請」等選項）。
2. **`templates/auth/login.html`** 與 **`register.html`**：
   - **繼承關係**：繼承 `base.html`，但使用無 Sidebar 的簡潔版面。
   - **說明**：美觀極簡的登入與註冊卡片。
3. **`templates/main/dashboard.html`**：
   - **繼承關係**：繼承 `base.html`。
   - **說明**：首頁主儀表板。店長版面採用 Grid 卡片式設計（缺工警示卡片呈紅色高亮、待審假單卡片呈橘色、今日出勤人數呈綠色）；員工版面則呈列本週個人班表與請假狀態。
4. **`templates/schedule/view.html`** 與 **`manage.html`**：
   - **繼承關係**：繼承 `base.html`。
   - **說明**：整合 FullCalendar.js 的月曆與週曆。`manage.html` 提供一鍵自動排班彈出表單與拖拉排班介面；`view.html` 為員工專用唯讀行事曆。
5. **`templates/leave/apply.html`** 與 **`review.html`**：
   - **繼承關係**：繼承 `base.html`。
   - **說明**：`apply.html` 提供員工請假表單（含日期選擇器）；`review.html` 提供店長表格化的假單審查，支援一鍵點擊 Approve / Reject 並填寫理由。
