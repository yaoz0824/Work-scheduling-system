# Work Scheduling System - 產品需求文件 (PRD)

## 1. 專案概述
- **背景與動機**：使用者需要一個可以紀錄個人上班時數、計算薪水與加班費的工具，以確保自身薪資權益並方便追蹤工作狀態。
- **目標用戶**：一般上班族、打工族、排班制人員。
- **核心價值主張**：提供一個簡單、直覺且符合勞動法規的工時與薪資計算工具，讓使用者對自己的收入與工時一目瞭然。

## 2. 功能需求

### 核心功能 (原需求)
1. **紀錄總時數**
   - 作為使用者，我希望能夠每天打卡或手動輸入上下班時間，以便系統幫我計算當日及當月總工時。
2. **計算薪水**
   - 作為使用者，我希望可以設定基本時薪或月薪，以便系統根據我的工時自動算出預估薪資。
3. **計算加班時數**
   - 作為使用者，我希望系統能自動判斷超時工作並標記為加班，以便計算加班費。

### 補充建議功能 (Nice to Have / 延伸功能)
4. **月曆視圖 (Calendar View)**
   - 作為使用者，我希望有一個月曆可以一覽每天的上班狀況、休假和時數，以便快速掌握整月的工作排程。
5. **符合勞基法之加班費試算**
   - 作為使用者，我希望加班費計算能支援勞基法規定（例如前兩小時 1.34 倍、後兩小時 1.67 倍，或休息日加班費率），以便得到準確的薪水預估。
6. **請假與特休紀錄**
   - 作為使用者，我希望可以標記請假（病假、事假、特休），以便系統在計算月薪時自動扣除對應比例。
7. **匯出報表功能 (Export)**
   - 作為使用者，我希望可以將當月工時紀錄匯出成 PDF 或 Excel，以便未來有勞資爭議時作為舉證，或個人記帳使用。
8. **多份工作管理**
   - 作為打工族，我希望可以建立多個工作設定（不同時薪、不同計算規則），以便同時管理多份工作的時數與收入。

## 3. 非功能需求
- **技術限制**：預計使用 Flask (Python) + Jinja2 + SQLite 進行開發 (根據專案預設技術棧)。
- **操作體驗**：需要有 RWD (響應式設計)，方便在手機上操作打卡或紀錄。
- **資料安全**：個人工時資料僅儲存於本地或綁定個人帳號，確保隱私。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**：手動新增每日上下班時間、總工時計算、單一工作的時薪/月薪計算、基本加班時數統計。
- **Should Have (應該有)**：月曆視圖、勞基法加班費率試算、簡單請假紀錄。
- **Nice to Have (可以有)**：多份工作管理、報表匯出功能、圖表統計(當月收入趨勢圖)。
